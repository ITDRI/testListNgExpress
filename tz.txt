Реализовать компонент, для отображения древовидной структуры по переданному в него списку объектов (компонент для дерева: https://material.angular.io/components/tree/overview).
Реализовать сервис, который является источником данных для компонента дерева, не принципиально откуда данные, может быть файл. Плюсом будет если данные разместить на json-server (https://www.npmjs.com/package/json-server) еще лучше что-то свое (бэк на nodejs).
 

Сущность элемента дерева:

{

            "id": 1,

            "parentId": null,

            "name": "o1"

        }

 

 

Поля id и parentId (тип number) используются для построения дерева, если parentId имеет значение null - корневой узел. Следует предусмотреть варианты с одним и несколькими корневыми узлами, так же - все узлы корневые.

Поле name (тип string) - произвольная строка.

 

При клике на узле дерева реализовать отображение дополнительной информации по объекту в правой части формы – еще один компонент (плюсом будет решение с ленивой загрузкой), отобразить дополнительную информацию в виде таблицы  "наименование - значение" (можно воспользоваться компонентом https://material.angular.io/components/table/overview).
Расширить сервис, на реализацию метода принимающего в качестве параметра идентификатор объекта и возвращающий расширенные свойства объекта, в формате:
 

{

    "id": 1,

    "description": "asdasdasd",

    "" ..... на что хватит фантазии, 3-4 поля

}
